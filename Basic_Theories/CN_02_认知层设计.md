# 认知层设计

**最新**: 2026-06-09  
**状态**: 正式发布  
**路径**: OpenAirymax/Docs/Basic_Theories/CN_02_认知层设计.md  

---

## 1. 引言

认知层是Airymax三层架构的顶层，负责将用户意图转化为可执行的任务计划，并协调 Agent 团队协同工作。其设计基于五维正交系统中的认知观，深受认知心理学和决策理论的启发，特别是双系统认知理论，以及"工程两论"（《工程控制论》和《论系统工程》）的反馈思想。本文档详细阐述认知层的理论根基、核心组件及运行机制。

## 2. 理论基础

### 2.1 双思考系统

双系统理论指出，人类思维由两个系统协同完成：

- **System 1（快系统）**：自动、直觉、低耗能，擅长模式匹配和快速响应。
- **System 2（慢系统）**：分析、推理、高耗能，用于复杂问题和规划。

Airymax 将这一模型映射到认知层的各个组件：

| 组件 | System 1 映射 | System 2 映射 |
|------|---------------|---------------|
| 路由层 | 快速意图分类 | 复杂度评估与资源匹配 |
| 双模型协同 | 辅模型快速验证 | 主模型深度推理 |
| 增量规划 | 快速响应已规划阶段 | 动态调整后续规划 |
| 调度官 | 快速候选筛选 | 多目标优化选择 |


这种分工使得认知层既能快速处理简单任务，又能在复杂场景下深度思考。

### 2.2 反馈控制论

《工程控制论》强调反馈是控制的核心。认知层的反馈闭环体现在：

- **实时反馈**：行动层执行结果通过 `execution_wait` 返回，影响当前任务的后续规划。
- **轮次内反馈**：增量规划器根据任务完成情况调整依赖关系。
- **跨轮次反馈**：进化委员会分析认知层统计信息，更新策略参数。

### 2.3 综合集成法

《论系统工程》提出的“从定性到定量的综合集成”方法在认知层体现为**多策略协同**：规划、协同、调度三类策略均可独立配置，由引擎动态组合，形成整体最优方案。

## 3. 核心组件

### 3.1 路由层

路由层是认知层的入口，负责将用户输入转换为结构化意图。其核心功能包括：

- **意图理解**：将自然语言解析为目标、约束、上下文。
- **复杂度评估**：根据关键词、长度等特征将任务分为简单、复杂、关键三类。
- **资源匹配**：根据复杂度选择对应的模型（轻量/最强）和 Token 预算。

### 3.2 双模型协同

双模型协同系统是认知层的“思维引擎”，实现 t1 快思考 与 t2 慢思考的协作：

- **主思考者（t2 慢思考）**：使用最强推理模型进行深度规划、反思和调整。
- **辅思考者（t1 快思考）**：使用轻量模型快速响应、交叉验证、冲突检测。
- **仲裁机制**：当主辅模型结论不一致时，触发内部辩论、审计委员会介入或人工确认。

### 3.3 增量规划器

增量规划器摒弃了一次性规划所有步骤的传统方法，而是：

- **分阶段规划**：只生成当前阶段的任务 DAG。
- **动态扩展**：根据执行反馈，逐步添加后续阶段任务。
- **回退与修正**：当任务失败时，可回退到前一个阶段，替换失败节点。

这种设计有效避免了长序列任务中的错误累积。

### 3.4 调度官

调度官负责从 Agent 注册中心选择合适的 Agent 执行任务。其核心是多目标优化：

$$
\text{Score}(agent) = w_1 \cdot \frac{1}{\text{cost}} + w_2 \cdot \text{success\_rate} + w_3 \cdot \text{trust\_score}
$$

权重可配置，支持成本优先、性能优先、信任优先等策略。
调度官还支持：

- **候选筛选**：根据任务角色（如 `product_manager`）过滤 Agent。
- **动态加载**：通过 `agent_pool` 按需加载 Agent 实例，避免资源浪费。

### 3.5 策略可插拔

认知层的三大策略（规划、协同、调度）均通过抽象接口定义，可在运行时动态替换：

- **规划策略**：分层规划、反应式规划、反思式规划、机器学习规划。
- **协同策略**：双模型协同、多数投票、加权融合、外部仲裁。
- **调度策略**：加权调度、轮询调度、优先级调度、机器学习调度。

这种设计使得系统可以针对不同场景优化，并允许社区贡献新策略。

## 4. 运行机制

### 4.1 处理流程

1. **用户输入** → 路由层解析意图 → 复杂度评估 → 资源匹配。
2. **主思考者生成初始规划** → 辅思考者交叉验证 → 若冲突则仲裁。
3. **增量规划器生成第一阶段任务 DAG** → 调度官为每个任务选择 Agent。
4. **行动层执行任务** → 执行结果反馈给认知层。
5. **增量规划器根据反馈调整后续规划** → 重复步骤 3-4 直至所有任务完成。
6. **结果汇总** → 返回给用户，同时记录统计信息（处理时间、Token 消耗等）。

### 4.2 健康检查

认知层提供健康检查接口，返回 JSON 状态，包括：

- 当前处理任务数、平均耗时、错误率。
- 各策略加载情况、可用模型列表。

这些信息可被监控服务采集，用于告警和自动扩缩。

## 5. 总结

认知层通过双系统理论、反馈控制、综合集成法，实现了智能体从“指令响应”到“主动规划”的跨越。其可插拔策略架构保证了系统的灵活性与可扩展性，为上层应用提供了稳定、高效的认知服务。
